## Summary

- close more of the Stage 1 `ContextAssembler` / `MemorySelection` ownership gap
- make TUI composer and compact responding behavior closer to Claude/Codex
- stabilize tool schema ordering and thread-exit resume hints

## What Changed

- context assembly
  - add an assembler-owned `AssembledTurnContext` so prompt/runtime views come from one turn contract
  - extend `MemorySelection` candidate selection and dropped-reason reporting without requiring vector retrieval
  - move TUI runtime snapshots closer to the runtime view by storing `memory_selection` as one object
- TUI behavior
  - make slash palette stop perturbing a manually scrolled transcript viewport
  - add multiline composer input with `Shift+Enter`
  - grow the bottom composer pane for multiline input
  - trim compact `Responding` output so internal `<plan>` / checklist blocks do not leak into the live response surface
  - ensure `/quit` prints the resume hint on a fresh line after terminal teardown
- tooling/runtime hygiene
  - make `ToolManager` schema ordering stable with `BTreeMap`
